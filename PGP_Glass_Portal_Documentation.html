3 Authentication Flow

1
User Opens Portal
Browser loads index.html. The login form is displayed with email + password fields. No session checks at this point.
2
Credential Validation
On form submit, handleFormLogin() looks up the submitted email + password against AppDB.getUsers() (localStorage). If no match â†’ error toast is shown + form shakes. If matched â†’ proceed.
3
Session Creation
Two values are written to localStorage:
• pgp_role = 'Super HR' or 'Branch HR'
• pgp_branch = 'All Locations' (Super HR) or the user's location (e.g., 'Kosamba') (Branch HR)
4
DB Initialization
AppDB.init() runs on every page load. It checks pgp_db_version against the current version (v2.0). If outdated, localStorage is flushed and re-seeded with fresh default data.
5
Navigation Guard
Every non-login page checks for a valid session on load. If pgp_role is missing â†’ redirect to index.html. If Branch HR tries to access a restricted page â†’ "Access Restricted" screen is rendered.
6
Shell Rendering
AppShell.renderSidebar() builds role-filtered nav. Super HR sees all 7 links. Branch HR sees: Dashboard, Apprentices, Reports, Settings only. AppShell.renderTopNav() shows branch selector (Super HR) or fixed branch pill (Branch HR).
7
Page Controller Init
The correct page controller is initialized based on URL path. Each controller fetches data, applies role-based filters, and renders the page UI.
